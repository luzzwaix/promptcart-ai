# PromptCart AI

PromptCart AI is a hackathon-ready autonomous merchant for Locus Paygentic Hackathon Week 4. LocusFounder starts the business; PromptCart AI runs it like a merchant.

Its thesis is simple:

> From one founder prompt, an AI agent invents a business, launches a storefront, accepts only profitable orders, spends its own wallet to manufacture the digital product, delivers it, and keeps operating with positive margin.

## Why This Is A Business, Not A Tool

Most AI products assist an operator. PromptCart AI becomes the operator.

For the canonical demo, LocusFounder generated the Pipeline Packs business from:

`Build a digital business that sells evidence-backed prospect packs to boutique CRO agencies.`

PromptCart then operates that Founder-created business:

- runs the business `Pipeline Packs`
- scores three possible business models before choosing the launch model
- chooses the market, ICP, pricing logic, and margin rule
- publishes sellable SKUs
- decides whether an order is commercially acceptable
- opens checkout only after that merchant judgment
- funds its own digital fulfillment pipeline
- ships a paid Prospect Pack
- remains online with operating metrics and next-SKU logic

That means the core artifact is not "AI-generated content." It is a working merchant loop.

## Canonical Economics

- Order: `Growth Pack`
- Price: `$99.00`
- Estimated fulfillment range: roughly `$6.70-$7.40`
- Realized simulated fulfillment spend: `$6.70`
- Gross margin after completion: about `93%`
- Wallet math after the canonical sale:
  - start: `$42.00`
  - payment inflow: `+$99.00`
  - fulfillment outflow: `-$6.70`
  - ending wallet: `$134.30`

The product intentionally shows both the merchant's pre-checkout judgment and the post-fulfillment unit economics.

## Why Locus Is Load-Bearing

Locus is not decorative here. The product loop depends on two commercial primitives:

1. **Checkout**  
   The order cannot proceed until the agent approves the economics and opens a checkout session.

2. **Merchant ledger spend**  
   The merchant credits checkout revenue into its simulated ledger, reserves fulfillment budget from that balance, spends against discrete production events, and settles the ledger after the digital product is manufactured.

Without those two primitives, PromptCart AI would collapse back into a research toy. With them, it becomes an autonomous merchant.

## Demo Mode Vs Production-Shaped Integration

This repository does **not** claim real payment processing or live Locus wallet settlement.

Checkout settlement is simulated because Stripe Connect is unavailable in Kazakhstan; Locus staff confirmed mock checkout is acceptable when no real payment is claimed.

### Real AI Fulfillment Proof Path

PromptCart includes one opt-in live Gemini fulfillment-agent call during the final synthesis stage of fulfillment. This is not presented as a Locus API claim; it proves the paid product can include real AI-powered manufacturing while the Locus Checkout and wallet settlement remain clearly simulated.

- Route: `src/app/api/locus/synthesize/route.ts`
- Live mode requires:
  - `GEMINI_API_KEY`
  - `ENABLE_LIVE_GEMINI_SYNTHESIS=true`
- AI fulfillment uses Gemini 2.5 Flash Lite with Gemini 2.5 Flash fallback, then deterministic fallback for demo stability.
- Live mode first calls the official Gemini API once:
  - `https://generativelanguage.googleapis.com/v1beta/models/gemini-2.5-flash-lite:generateContent`
  - model: `gemini-2.5-flash-lite`
- If the primary model fails with a non-200 response, missing candidate text, malformed JSON, or a network error, the route tries:
  - `gemini-2.5-flash`
- The call produces:
  - agent decision
  - selected accounts
  - pack summary
  - quality gate
  - outreach opener
  - operator note

If the key or flag is absent, or if the call fails, the route returns deterministic fallback agent output and the canonical demo flow remains stable. The API key is never exposed to the client, and no live call is made on page load.

## Architecture

Two-part architecture:

1. LocusFounder created the business.
2. PromptCart operates the business.
3. Checkout and wallet behavior use simulated rails mapped to Locus primitives.

- `src/lib/types.ts`
  - explicit domain model:
    - `FounderPrompt`
    - `BusinessBlueprint`
    - `SKU`
    - `Order`
    - `ProfitabilityAssessment`
    - `FulfillmentJob`
    - `SpendEvent`
    - `ProspectLead`
    - `DeliverablePack`
    - `LocusCheckoutGateway`
    - `LocusWalletClient`
- `src/lib/business-engine.ts`
  - business selection logic
  - SKU/pricing generation
  - profitability evaluator
  - merchant accept/reprice/reject logic
  - fulfillment state machine
  - spend and margin math
- `src/lib/locus-adapters.ts`
  - production-shaped checkout and wallet ports
  - demo implementations
- `src/components/promptcart-app.tsx`
  - interactive merchant control room
  - local demo persistence
  - checkout flow
  - fulfillment animation
  - deliverable export

The demo state persists in `localStorage`, with a versioned envelope so stale browser state does not corrupt the judging flow.
